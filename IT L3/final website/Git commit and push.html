To commit and push your project (like the one shown in your screenshot) to a GitHub repository, follow these steps using your terminal or VS Code's built-in terminal:

✅ 3. Stage Your Files

bash
git add .

This adds all files to be committed.

✅ 4. Commit Your Changes

bash
git commit -m "Initial commit of final website"

✅ 5. Push to GitHub

If this is your first push to a new repo:

bash
git branch -M main
git push -u origin main

Otherwise, for subsequent pushes:

bash
git push

You can also do this via GUI:

The error message in your terminal says:

❌ Updates were rejected because the remote contains work that you do not have locally.

This means:
Your GitHub repo (finalwebsiteassessment2024) already has files/commits (maybe a README or .gitignore) — but your local folder doesn’t. So Git is blocking the push to prevent accidental overwrite.

✅ Solution: Pull First, Then Push

Run the following safe commands:

bash
git pull origin main --allow-unrelated-histories

This merges your local project with whatever is already in the GitHub repo.

Then, you can push:

bash
git push -u origin main

🔐 If It Asks for Authentication:

Since you're using HTTPS (e.g., https://github.com/username/...), Git may ask for a GitHub login token. Here’s how to fix it if it fails:

Option 1: Use GitHub CLI (Recommended)

Install GitHub CLI: https://cli.github.com/
Then run:

bash
gh auth login

Option 2: Use Personal Access Token (PAT)

If it asks for username/password:

  • Use your GitHub username.

  • Use a GitHub Personal Access Token as your password (not your GitHub password!).
